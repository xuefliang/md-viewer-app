# 翻译视图修复设计

## 问题背景

在 MD Viewer 的「翻译」视图中存在三个问题：

1. **右侧滚动条不可见**：翻译内容超出可视区域后无法明显看到滚动条，影响长文档浏览。
2. **翻译完成后会再次翻译**：每次切换到翻译视图都会无条件重新调用翻译；翻译完成后状态仍停留在「翻译中... 块 2/16」之类的显示，并会再次触发翻译流程。
3. **不会自动保存翻译结果**：目前翻译完成后仅显示「保存翻译」按钮，需要用户手动点击保存；希望全文翻译完成后自动保存为 `.md` 文件。

## 设计方案

### 1. 修复滚动条可见性

- 为 `#translate-view` 增加显式滚动样式：
  - `overflow-y: auto`
  - `height: 100%`
  - 补全 WebKit 滚动条 `::-webkit-scrollbar` 与 `::-webkit-scrollbar-thumb` 样式，使右侧滚动条在翻译视图中可见。
- 保持 `#reader-content` 的滚动容器角色，但让 `translate-view` 在翻译模式下独立滚动，避免与编辑器/预览区域混用滚动行为。

### 2. 避免重复翻译

- 修改 `setViewMode()` 中切换到 `translate` 视图的逻辑：
  - 如果当前标签页已存在 `translatedContent`，则不再调用 `startTranslation()`，直接显示已有结果。
  - 如果当前标签页没有 `translatedContent`（首次切换），才调用 `startTranslation()`。
- 翻译成功后设置 `tab.translatedContent`，作为缓存依据。
- 保留「保存翻译」按钮作为手动保存入口；后续如需「重新翻译」可再增加按钮，本次不在范围内。

### 3. 翻译完成后自动保存 `.md`

- 在 `startTranslation()` 的完成分支中，当翻译成功且原文有文件路径时：
  - 自动生成目标路径：`原文件路径` → `原文件名.translated.md`（与原文件同目录）。
  - 调用后端 `write_markdown_file` 将 `tab.translatedContent` 写入该路径。
  - 将保存结果通过右下角状态提示或进度区文本反馈给用户（成功显示「翻译已保存」；失败显示错误原因）。
- 如果原文是新建草稿（无 `tab.path`），则不自动保存，仍依赖手动「保存翻译」按钮。
- 保存过程不应阻塞 UI；如保存失败，仅提示错误，不影响已渲染的翻译结果。

## 涉及文件

- `src/styles.css`：修复滚动条样式。
- `src/main.js`：
  - `setViewMode()` 增加已有翻译结果的判断。
  - `startTranslation()` 完成后触发自动保存。
- `src/i18n.js`：新增/调整自动保存相关文案（可选，复用现有 `translate.saveSuccess`/`translate.saveFailed`）。

## 验收标准

- [ ] 翻译视图右侧出现可操作的滚动条。
- [ ] 翻译完成后切换回翻译视图不会重新翻译，进度条不再回到「翻译中...」。
- [ ] 翻译完成后，同目录下自动生成 `<原文件名>.translated.md`。
- [ ] 自动保存失败时给出错误提示，成功时给出简短成功提示。
- [ ] 现有「保存翻译」手动按钮仍然可用。
